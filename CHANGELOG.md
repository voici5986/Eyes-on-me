# 变更记录

## 2026-03-27

### 桌面采集端

- 增加 `presence` 状态：`active`、`idle`、`locked`
- 增加锁屏检测和空闲检测，避免把无人使用或锁屏时段继续算进活跃时长
- 采集模式从纯切换驱动改为混合模式：
  - 前台切换时立即上报
  - 每 15 秒补样一次
- macOS 恢复基于系统通知的前台应用切换监听
- 降低 macOS 高频重型 AppleScript 调用频率
- 扩展浏览器识别、URL 推断和域名提取能力

### 服务端与分析

- SQLite 持久化中增加 `presence` 字段
- 新增浏览器分层聚合：
  - 浏览器
  - 域名
  - 页面
- 避免旧事件之间超长时间间隔被错误累计成连续使用时长
- 最近活动流中过滤掉噪声较大的采样事件，保留更可读的切换记录
- 新增 `today` 时间范围，按服务端本地时区自然日 00:00 到当前时间统计，不再等同于最近 24 小时

### Web 页面

- 页面文案和状态展示支持空闲、锁屏状态
- 首页和设备分析页增加浏览器 / 域名 / 页面三级分析视图
- 域名卡片支持展开页面级明细，查看同一域名下各页面分别停留了多久
- 修复“正在使用多久”误显示为超长连续时长的问题，改为显示最近上报时效
- 优化域名分析卡片中右侧占比列的布局
- 时间范围筛选新增“今天”

### 打包与开发

- 新增 `run-web-dev.sh`，支持本地直接启动 Vite 开发服务器
- 新增 `run-dev.sh`，用于快速查看本地开发启动方式
- bundle 中的 agent 默认静默读取配置，不再每次交互提示
- bundle 启动脚本统一在 bundle 根目录执行
- `client-server` 现在会在构建时直接内嵌 `web/dist`，发布包不再依赖额外复制的 `web/dist`
- 打包默认保留 bundle 目录中原有的数据库
- 只有显式指定 `PACKAGE_COPY_DB=1` 时，才会把根目录数据库复制进 bundle
- GitHub Actions 已同步调整为先构建前端，再编译服务端，保证发布产物内嵌的是最新页面

### 后续待办

- macOS 采集仍建议逐步从 AppleScript 迁移到原生 API / Accessibility 方案
